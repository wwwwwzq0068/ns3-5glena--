# 毕设中期汇报技术总结

## 1. 汇报主线
当前中期汇报不宜写成“改了很多脚本”，而应明确为三层成果：

1. 建立了面向 LEO-NTN 切换研究的三维物理模型。
2. 在此基础上整理出可复现、可统计、可扩展的仿真平台。
3. 在平台上收口了当前阶段的传统 `A3 baseline`，并为后续联合策略预留了统一入口。

建议的汇报顺序：
- 先讲课题目标和当前阶段定位
- 再讲三维物理模型与 `Earth-fixed` 覆盖设计
- 然后讲当前 baseline 平台
- 最后讲不足与下一步优化方向

## 2. 课题目标与当前阶段定位

### 2.1 课题目标
本课题研究的是低轨卫星高速运动场景下的 `5G-NTN` 切换问题，核心关注点不是链路能否建立，而是：
- 卫星快速过境时，地面 `UE` 如何平稳切换
- 切换前后业务连续性如何保持
- 传统基于信号强度的切换方法会暴露哪些问题
- 后续如何引入“信号质量 + 卫星负载”的联合决策

### 2.2 当前阶段定位
当前研究仓库最近已发布稳定节点为 `3.2.1`（Git tag：`research-v3.2.1`）。这里的 `3.2.x` 是研究工作版本，不是 ns-3 框架版本；底层仿真框架仍为 `ns-3.46`。当前工作区仍在 `3.2` 主阶段内继续整理 `seven-cell baseline`，但尚未单独提升新的稳定节点。

当前阶段的重点已经从“先把基础物理场景构建出来”推进到“将其整理为可用于后续毕设实验的 baseline 平台”。这意味着：
- 不再把继续扩星作为默认主线
- 优先固定 `2x4` 双轨场景和传统 `A3` baseline
- 为后续联合策略保留明确的对照组和接入口

## 3. 三维物理模型设计

### 3.1 为什么必须采用三维模型
如果只使用二维近似模型，很难正确反映：
- 卫星相对地面的真实过境
- 仰角、斜距和最大覆盖的时变关系
- 多普勒频移的连续演化
- 轨道倾角和不同轨道面对星地几何的影响

因此，本课题前期首先完成了从简化平面模型到三维物理可解释模型的升级。

### 3.2 WGS84 地理锚定
地面 `UE` 位置采用 `WGS84` 地球模型中的经纬高描述，再映射到地心坐标。这样做的意义是：
- 使 `UE` 位置具有真实地理含义
- 让星地距离、仰角、方位角和后续地面网格都建立在统一几何基础上

当前默认地面参考点为：
- 纬度：`45.6°`
- 经度：`84.9°`
- 高度：`0 m`

### 3.3 ECI 与 ECEF 的分工
当前模型采用两层坐标描述：
1. 在 `ECI`（地心惯性系）中计算卫星轨道运动
2. 在 `ECEF`（地心地固系）中分析卫星相对地面的覆盖与服务关系

可以简单理解为：
- `ECI` 用来回答“卫星本身怎么飞”
- `ECEF` 用来回答“卫星相对地面用户在哪里”

### 3.4 关键几何量
在得到卫星 `ECEF` 位置和速度之后，当前平台可计算：
- 斜距
- 仰角
- 方位角
- 多普勒频移

这些量分别服务于：
- 传播损耗分析
- 可见性与切换候选判定
- 用户天空方位解释
- 高动态 `NTN` 频率偏移分析

### 3.5 轨道对齐与场景可控性
为保证仿真可解释、可重复，当前模型引入了轨道对齐与时序控制参数，包括：
- `alignmentReferenceTimeSeconds`
- `overpassGapSeconds`
- `interPlaneTimeOffsetSeconds`

这些参数的作用是让多颗卫星与目标区域在仿真时间轴上形成可预期的过境关系，使切换事件来自可控几何，而不是随机碰撞。

## 4. Earth-fixed 覆盖与小区设计

### 4.1 为什么需要 Earth-fixed
如果波束始终追着单个 `UE` 跑，虽然链路看起来稳定，但会弱化两个研究重点：
- 波束边缘增益滚降和候选星竞争
- 固定地面小区语义下的切换分析

因此，当前平台采用 `Earth-fixed` 设计：先在地面定义固定服务区域，再让运动中的卫星通过波束去服务这些区域。

### 4.2 波束与小区的关系
- 波束是物理层概念，表示空间中的定向覆盖
- 小区是网络层概念，表示协议栈中的逻辑服务单元

当前设计中，两者的关系是：
- 地面先有固定区域
- 卫星通过波束覆盖这些区域
- `UE` 看到的是服务小区或服务卫星变化，而不是单纯的“光斑扫过”

### 4.3 基于 WGS84 的局部六边形网格
当前平台不做全球网格，而是在研究区域内构造局部 `WGS84` 六边形网格：
- 区域范围：`400 km x 400 km`
- 六边形半径：`20 km`
- 最近候选数：`K = 3`

这些网格中心作为候选地面锚点，使 `Earth-fixed` 设计具备明确的数据结构和运行路径。

### 4.4 设计价值
`Earth-fixed` 设计的价值主要在于：
- 让覆盖建模具备网络规划语义
- 让切换分析更容易映射到固定地面区域
- 为后续资源管理和联合决策提供地理单元基础

## 5. 当前 baseline 平台

### 5.1 baseline 的研究定位
当前阶段的 baseline 不是最终优化算法，而是对照组。它的作用是：
- 作为传统切换方法的参考系统
- 暴露高动态双轨场景下传统 `A3` 的局限
- 为后续改进策略提供统一对比口径

### 5.2 当前主场景
根据当前稳定节点 `3.2.0`，baseline 主场景为：
- 卫星数：`8`
- 轨道面数：`2`
- 轨道结构：`2x4`
- `UE` 数：`25`
- `UE` 部署：中心小区 `3x3` 密集簇 `9 UE`，外围 `6` 个相邻小区共 `16 UE`
- hex 小区半径：`20 km`
- 中心簇间距：`6 km`
- 外围小区内局部散点偏移：`5 km`
- 轨道高度：`600 km`
- 轨道倾角：`53°`
- 轨道面 `RAAN` 间隔：`3°`
- 轨道面时间偏移：`0.3 s`
- 对齐参考时刻：`20 s`
- 同轨过境间隔：`2 s`
- 仿真时长：`40 s`

这套配置的目标不是追求最大规模，而是用可控场景增强跨轨竞争、多用户空间差异和局部负载聚集。

### 5.3 当前切换机制
当前 baseline 采用传统 `A3` 风格切换语义，主要依据为：
- `RSRP`
- `hysteresis`
- `TTT`
- 邻区可达性约束

当前默认切换参数：
- `hoHysteresisDb = 3.0 dB`
- `hoTttMs = 300 ms`
- `customA3ShadowingSigmaDb = 1.0 dB`
- `customA3ShadowingCorrelationSeconds = 4.0 s`
- `customA3RicianKDb = 15 dB`
- `customA3RicianCorrelationSeconds = 1.0 s`

此外，当前平台保留 `strictNrtGuard`（严格邻区表守卫）这一增强开关，但当前 baseline 默认不启用；baseline 只保留基本的可见性与 `beam lock` 约束，用于更真实地暴露传统 `A3` 的局限。

### 5.4 baseline 的意义
当前 baseline 的目标不是证明传统 `A3` 很优，而是提供一个清楚、稳定、可统计的起点。只有 baseline 定义清楚，后续改进策略的收益才有对照意义。

## 6. 平台化与工程化整理

### 6.1 模块拆分
当前主要仿真入口仍是：
- `scratch/leo-ntn-handover-baseline.cc`

但关键功能已经拆分到独立头文件中，例如：
- `leo-ntn-handover-config.h`
- `leo-ntn-handover-runtime.h`
- `leo-ntn-handover-update.h`
- `leo-ntn-handover-reporting.h`
- `leo-ntn-handover-utils.h`

这样做的意义是让主脚本更聚焦于场景搭建和流程控制，同时降低后续算法扩展时的结构阻力。

当前 `UE` 布局生成也做了一次同样思路的收口：
- 先生成局部东-北平面中的七小区布局偏移模板
- 再统一转换为 `WGS84` 地理点和 `ECEF` 位置

这样做的意义是把“布局规则”和“地理坐标映射”解耦，后续如果要继续调七小区内部的 `UE` 分布，不需要改动主转换链路。

### 6.2 运行时状态组织
当前平台已经对卫星侧和 `UE` 侧运行时状态做了结构化管理。

卫星侧重点包括：
- 当前锚点与邻区集合
- 已接入 `UE` 数
- 汇总业务到达率
- `loadScore`
- `admissionAllowed`

`UE` 侧重点包括：
- 初始接入与当前服务关系
- 切换开始与成功计数
- 切换执行时延累计
- 自定义切换逻辑中的候选与服务状态

### 6.3 日志与结果输出
当前平台已统一支持：
- 切换开始与成功日志
- 服务星变化日志
- 周期性仿真进度输出
- 最终吞吐统计和切换汇总
- `sat_beam_trace` 等逐时刻导出

结果目录统一为：
- `scratch/results/`

这样可以减少结果散落，便于后续做 baseline 与改进算法对照。

需要说明的一点是：
- 当前 `PHY` 信道已保留 `ThreeGpp` 路径并开启 `ShadowingEnabled`
- 但 baseline 的 `A3` 判决主线仍使用自定义几何 `beam budget/rsrpDbm`，还没有把随机衰落直接接进判决链
- 当前平台已经补上一个可开关实验入口：可将 `shadowing / Rician` 扰动注入 custom `beam budget/A3` 观测链；当前默认已开启，但判决仍不是直接读取 PHY 测量

### 6.4 稳定性与性能处理
已完成的非算法性关键工作包括：
- 周期性进度输出
- 几何共享计算优化，减少重复轨道传播
- 多处切换后协议异常防御
- 默认关闭 `UE IPv4 forwarding`，避免异常包被 UE 二次转发回 `NAS/TFT` 分类链
- 保留 `forceRlcAmForEpc` 作为可选稳定性开关，用于单独比较 `RLC AM/UM` 在切换附近的表现
- 关闭与当前 handover 主线无关的高噪声输出
- 默认关闭容易触发无关 `PHY fatal` 的部分 `SRS` 项

这些工作虽然不直接等于“算法创新”，但决定了平台能否稳定支撑后续实验。

### 6.5 为联合策略预留接口
当前 baseline 仍然是纯信号质量驱动，但平台已经预留了负载相关接口：
- `attachedUeCount`
- `offeredPacketRate`
- `loadScore`
- `admissionAllowed`

这意味着后续扩展不需要推翻平台，只需要在现有切换链路中加入联合决策模块。

## 7. 当前阶段成果、不足与下一步

### 7.1 当前已完成的核心成果
可以归纳为四项：
- 完成了三维物理模型设计
- 完成了 `Earth-fixed` 覆盖建模思路与局部落地
- 完成了当前 `A3 baseline` 的主场景和对比口径收口
- 完成了平台化与工程化整理

### 7.2 当前仍存在的不足
当前平台已经具备 baseline 研究能力，但仍有三类不足：
- baseline 仍需通过更多实验确认其问题暴露是否稳定
- 负载接口已接入，但尚未真正进入切换决策
- 随机信道扰动与自定义判决链之间仍有一层语义断点，需要先单独澄清后再推进增强策略

### 7.3 下一步优化方向
下一阶段建议分三条线推进：

1. baseline 验证线
   - 冻结并验证 `seven-cell baseline` 主配置
   - 观察频繁切换、潜在 `ping-pong` 和吞吐连续性
   - 核实卫星过境与七小区占位关系是否稳定可解释

2. 信道扰动澄清线
   - 独立评估 `shadowing / Rician` 扰动是否接入自定义 `beam budget/A3` 判决链
   - 区分“PHY 更真实”与“切换判决已受扰动影响”这两个层次
   - 在不改变 baseline 默认定义的前提下，确认边界竞争与 `ping-pong` 会如何变化

3. 改进策略准备线
   - 在当前平台上正式引入负载感知
   - 设计“信号质量 + 卫星负载”的联合指标
   - 在统一 baseline 对照下验证收益

## 8. 适合中期汇报的结论表达
中期汇报结论可以概括为：

1. 已完成面向 `LEO-NTN` 切换研究所需的三维物理模型设计，包括 `WGS84` 地理锚定、`ECI/ECEF` 转换及关键星地几何量计算。
2. 已完成 `Earth-fixed` 覆盖设计思路和局部六边形地理网格建模，为固定地面小区研究奠定了基础。
3. 当前工作重点已从基础模型构建转向平台化与 baseline 收口，已经形成了以 `2x4` 双轨、`25 UE` 和传统 `A3` 风格切换为核心的可运行 baseline 平台。
4. 平台已具备模块化结构、统一输出、运行时状态管理和后续联合策略扩展接口，能够支撑后续毕设实验。
5. 下一阶段将围绕 baseline 验证和“信号质量 + 卫星负载”联合决策优化继续推进。

## 9. 汇报时的简要讲法
推荐口头顺序：
- 先说明课题目标是研究 `LEO-NTN` 切换，不是普通连通性
- 再说明前期先解决三维物理建模问题
- 然后说明在此基础上引入 `Earth-fixed` 覆盖和地面小区设计
- 接着说明 `2.5` 之后重心转向 baseline 平台搭建与工程化整理
- 最后说明当前 baseline 的配置、作用以及下一步改进方向
