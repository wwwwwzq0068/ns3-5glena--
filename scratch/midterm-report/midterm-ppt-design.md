# 毕设中期答辩 PPT 设计方案

本文件用于沉淀当前中期答辩 `PPT` 的推荐结构、每页放置内容、配图建议与排版重点，便于后续直接照此制作幻灯片。

当前口径与本方案保持一致：
- 研究主线：`LEO-NTN` 切换研究
- 当前 baseline：`2x4` 双轨、`25 UE`、`seven-cell`
- 当前最新稳定节点：`research-v4.2`
- 当前阶段定位：完成三维物理建模、`Earth-fixed` 覆盖建模与统一真实测量驱动的 `A3 baseline / improved` 平台收口
- 当前重点：在同一 `MeasurementReport` 入口下比较 baseline 与 improved

---

## 1. 使用建议

- 推荐页数：`14` 页主 `PPT` + `1` 页参数备用页
- 当前推荐目录共 `5` 个部分
- 汇报主线：`研究背景与意义 -> 课题目标 -> 当前技术与平台 -> 初步结果分析 -> 下一步工作`
- 每页只表达一个核心结论
- 每页正文控制在 `3-5` 条以内
- 图优先于长段文字
- 不建议直接放代码截图、终端日志截图、原始 `CSV` 截图

建议贯穿全篇的总述句：

> 本课题不是单纯搭建仿真程序，而是面向 `LEO-NTN` 切换研究，构建一个可复现、可对照、可扩展的 baseline 平台，并在此基础上推进联合切换策略设计。

## 第一部分：研究背景与意义

## 第 3 页：研究背景

### 页面标题
`研究背景`

### 建议文字
- `5G/6G` 网络正向空天地一体化方向发展，`NTN` 是重要组成部分
- `LEO` 卫星具备广域覆盖和较低时延优势，可补充地面网络覆盖
- 但 `LEO` 卫星高速运动、覆盖窗口短、服务关系变化快
- 因此 `LEO-NTN` 场景中的切换频率更高、切换问题更复杂

### 配图建议
- 一张“多颗 `LEO` 卫星掠过地面 `UE`”的问题示意图
- 图中可体现卫星移动、覆盖区域变化、服务切换箭头

### 图注建议
`图 1 LEO-NTN 高动态切换场景示意图`

### 版式建议
- 左侧放 `4` 条背景文字
- 右侧放问题示意图

### 页面标题

## 第 8 页：Earth-fixed 覆盖与场景建模

### 页面标题
`Earth-fixed 覆盖与场景建模`

### 建议文字
- 不采用波束持续追踪单个 `UE` 的建模方式
- 采用 `Earth-fixed` 思想，先定义固定地面服务区域
- 再由运动卫星通过波束对这些区域提供服务
- 当前研究区域采用局部六边形网格组织覆盖关系

### 配图建议
- 优先使用“六边形网格 + UE 分布”叠加图

建议中期答辩始终围绕以下逻辑展开：

1. 为什么要研究 `LEO-NTN` 切换
