# Scratch 目录说明

## 目录用途
- 本目录存放当前毕设中使用的 LEO 切换仿真代码、辅助头文件和分析脚本。
- 当前主要的仿真入口文件是 `scratch/leo-ntn-handover-baseline.cc`。
- 当前研究仓库版本定义为 `3.0`。
- 这里的 `3.0` 是研究工作版本，不是 ns-3 框架版本；ns-3 本身仍然是 `3.46`。

## 当前基础组目标
- 当前阶段先构建基础组，再在此基础上设计优化组。
- 当前 `3.0` 阶段优先扩大基础组的星座规模，再在更有代表性的基础组上扩展优化组。
- 当前基础组目标为：
  - 5 颗同轨道 LEO 卫星
  - 3 个 UE
- 目前 `leo-ntn-handover-baseline.cc` 中已经确认的实现状态：
  - 卫星数量已经配置为 5
  - UE 数量已经改为可配置，默认值为 3
  - UE 采用简单的地面间隔模型进行部署
- 当前阶段说明：
  - 文件拆分和代码结构整理已经完成当前阶段目标
  - 切换日志与最终汇总格式已经完成当前阶段整理
  - 下一步应优先把场景从 5 星局部过境扩展到更大星座规模

## 当前优先任务
- 逐步扩展星座规模，使场景不再局限于 5 星局部过境。
- 检查星座扩展后邻区管理、可见性门控和自定义 A3 执行器是否仍然稳定。
- 在更大星座下继续关注切换相关结果，避免日志量失控。
- 评估扩大星座规模后对切换成功率、执行时延和吞吐连续性的影响。
- 增加适量中文注释，但不再把结构整理作为当前主任务。

## 日志输出偏好
- 默认更关注以下内容：
  - 切换开始
  - 切换成功或失败
  - 服务卫星变化
  - 最终统计结果
- 日常调试时尽量减少低价值噪声日志。

## 结果输出目录
- 当前默认仿真输出统一写入 `scratch/results/`。
- 现阶段以下文件会优先落到该目录：
  - `hex_grid_cells.csv`
  - `sat_beam_trace.csv`
  - `sat_attenuation_per_time.csv`
- 该目录默认被 `.gitignore` 忽略，不直接进入版本库。
- 需要长期保留的基线结果，后续应单独挑选后再纳管，而不是把全部临时输出直接提交。

## 当前分析脚本
- `sat_attenuation_report.py`
  - 默认读取 `scratch/results/sat_beam_trace.csv`
  - 默认生成 `scratch/results/sat_attenuation_per_time.csv`
- `plot_hex_grid_svg.py`
  - 读取六边形网格 CSV
  - 生成对应的 SVG 可视化图

## 维护规则
- 每次对 `scratch/` 目录下的重要代码做出修改后，都同步更新本文件。
- 记录内容保持简洁、明确，重点写清：
  - 改了什么
  - 为什么改
  - 是否影响基础组行为
- 版本与分支、提交命名规则以 `docs/research-workflow.md` 为准。
